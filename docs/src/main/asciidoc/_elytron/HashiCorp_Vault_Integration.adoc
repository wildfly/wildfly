[[Vault_Integration]]
= HashiCorp Vault Integration

To use credential stores backed by HashiCorp Vault in the WildFly management omdel, the Galleon HashiCorp Vault feature pack can be used. This feature pack adds the `hashicorp-vault` subsystem to WildFly, which allows you to define Elytron credential store resources that read secrets from a HashiCorp Vault server. It also allows you to define expressions backed by HashiCorp vault.

There are several ways to install the HashiCorp Vault feature pack. Refer to the Galleon documentation for these.

== HashiCorp Vault credential store

After the feature pack is installed, define one or more credential store resources in the `hashicorp-vault` subsystem. You can then reference credentials from these stores elsewhere in the configuration using the standard `credential-reference` pattern with `store` and `alias`.

Aliases use the form _`<vault-path>.<key>`_: the path to the secret in Vault (e.g. `secret/database`) followed by a dot and the key name (e.g. `password`), giving `secret/database.password`.

=== Credential references backed by HasihCorp Vault

Use the credential store name and alias in any resource that supports `credential-reference` (e.g. datasources, Elytron key stores, etc.):

[source,xml,options="nowrap"]
----
<credential-reference store="my-vault" alias="secret/database.password"/>
----

=== Encrypted expressions backed by HasihCorp Vault

You can reference HashiCorp Vault credential store anywhere where the expressions are allowed in the management model. Pattern for the expressions is:

[source,options="nowrap"]
----
${HC_VAULT::hashicorp-vault-credential-store-name:alias}
----

In use for example with credential reference:

[source,options="nowrap"]
----
<credential-reference={clear-text=${HC_VAULT::my-vault:secret/myroot.myroot}}/>
----
